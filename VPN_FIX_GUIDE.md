# Решение проблемы с VPN

## Проблема

- С VPN включенным: проблемы с локальными подключениями (Ollama, Backend)
- С VPN отключенным: не работает интернет

## Возможные причины

1. **VPN блокирует локальные подключения** (localhost, 127.0.0.1)
2. **VPN меняет DNS настройки** - после отключения DNS не восстанавливается
3. **VPN меняет маршрутизацию** - после отключения маршруты не восстанавливаются

## Решения

### 1. Проверьте DNS настройки

После отключения VPN:
```bash
# Очистить DNS кэш
ipconfig /flushdns

# Проверить DNS
ipconfig /all

# Установить DNS вручную (если нужно)
netsh interface ip set dns "Ethernet" static 8.8.8.8
netsh interface ip add dns "Ethernet" 8.8.4.4 index=2
```

### 2. Проверьте маршрутизацию

```bash
# Показать маршруты
route print

# Удалить маршруты VPN (если остались)
route delete 0.0.0.0 mask 0.0.0.0 <VPN_GATEWAY>
```

### 3. Настройте VPN для локальных подключений

В настройках VPN:
- Отключите "Kill Switch" (если есть)
- Отключите "Block local network" (если есть)
- Добавьте исключения для localhost/127.0.0.1

### 4. Временно отключите VPN для тестирования

1. Отключите VPN
2. Очистите DNS: `ipconfig /flushdns`
3. Проверьте интернет: `ping google.com`
4. Если не работает, установите DNS вручную

## Проверка

После отключения VPN:
```bash
# Проверить интернет
ping google.com

# Проверить DNS
nslookup google.com

# Проверить локальные подключения
ping 127.0.0.1
```
