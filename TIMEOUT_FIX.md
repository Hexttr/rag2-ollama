# ✅ Исправление проблемы таймаутов

## Проблема

**Request timed out** - запросы к Ollama превышают timeout (300 секунд = 5 минут)

Из логов:
- Первые запросы успешны
- Потом начинаются таймауты каждые 5 минут
- Retry каждые 5 минут
- Все 10 попыток исчерпаны
- Процесс зависает

## Исправления

### 1. Увеличен timeout

**Файлы:**
- `backend/app/core/config.py`: `OLLAMA_TIMEOUT: 300` → `900` (15 минут)
- `pageindex_ollama.py`: `timeout=300` → `timeout=900` (во всех местах)
- `backend/app/services/pageindex_service.py`: `timeout=300` → `timeout=900`

**Причина:**
- Для больших документов Ollama может обрабатывать запросы дольше 5 минут
- 15 минут должно быть достаточно для большинства случаев

### 2. Что дальше

После применения исправлений:
1. Перезапустить backend
2. Удалить документ ID 2 (застрявший)
3. Загрузить документ заново
4. Индексация должна работать без таймаутов

## Готовность

✅ **Timeout увеличен до 15 минут**
✅ **Исправления применены**

Теперь индексация должна работать без таймаутов.
